# TOTEM ZMK Firmware APS GOOD

## Назначение репозитория

Данный репозиторий содержит пользовательскую конфигурацию ZMK для клавиатуры TOTEM на базе контроллера XIAO BLE.

Репозиторий предназначен для:

- автоматической сборки прошивок через GitHub Actions;
- получения файлов прошивки для левой и правой половины клавиатуры;
- получения прошивки для донгла;
- получения отдельной прошивки `settings_reset`;
- автоматической генерации рисунка раскладки на основе keymap.

---

## Что генерируется в репозитории

При запуске workflow собираются следующие варианты прошивок:

### 1. Левая половина, режим Bluetooth

Артефакт:

- `totem_qwerty_left.uf2`

Назначение:

- прошивка для левой половины клавиатуры при прямом подключении по Bluetooth.

### 2. Правая половина, режим Bluetooth

Артефакт:

- `totem_qwerty_right.uf2`

Назначение:

- прошивка для правой половины клавиатуры при прямом подключении по Bluetooth.

### 3. Левая половина, режим dongle

Артефакт:

- `totem_qwerty_dongle_left.uf2`

Назначение:

- прошивка для левой половины при использовании клавиатуры через донгл.

### 4. Правая половина, режим dongle

Артефакт:

- `totem_qwerty_dongle_right.uf2`

Назначение:

- прошивка для правой половины при использовании клавиатуры через донгл.

### 5. Донгл

Артефакт:

- `totem_qwerty_dongle.uf2`

Назначение:

- прошивка USB-донгла.

### 6. Донгл с экраном

Артефакт:

- `totem_qwerty_dongle_yads.uf2`

Назначение:

- прошивка донгла с модулем отображения статуса.

### 7. Донгл Prospector

Артефакт:

- `totem_qwerty_dongle_prospector.uf2`

Назначение:

- специальный вариант прошивки донгла с модулем `prospector_adapter` и `studio-rpc-usb-uart`.

### 8. Сброс настроек

Артефакт:

- `firmware_reset_totem.uf2`

Назначение:

- отдельная сервисная прошивка для очистки Bluetooth-настроек и пользовательских параметров.

## Какие файлы используются для сборки

Основные файлы конфигурации:

- `build.yaml` — матрица сборки;
- `config/west.yml` — список подключаемых модулей;
- `config/totem.keymap` — исходная keymap-конфигурация;
- `boards/shields/totem-bt/*` — настройки shield для Bluetooth-варианта;
- `boards/shields/totem-dongle/*` — настройки shield для варианта с донглом;
- `.github/workflows/build.yml` — основной workflow GitHub Actions;
- `.github/workflows/user_config_build.yaml` — reusable workflow сборки;
- `.github/workflows/draw_keymaps.yaml` — reusable workflow построения рисунка раскладки.

---

## Как устроена раскладка

В текущей конфигурации используются следующие логические слои:

- `BASE` — основной слой;
- `Navigation` — навигация, стрелки, мышь, page up/page down;
- `digits` — цифры и специальные символы;
- `sim_layer` — системный слой, Bluetooth, output, мультимедиа.

Дополнительно настроены:

- homerow-модификаторы;
- комбо Backspace;
- комбо альтернативного Backspace;
- комбо Shift;
- комбо перехода к цифровому слою;
- комбо Caps Word;
- макросы для переключения языка;
- макросы для специальных символьных последовательностей.

---

## Переключение языка

В прошивке предусмотрены комбо для отправки системных сочетаний переключения раскладки операционной системы.

Используются макросы:

- `lang_en_os`
- `lang_ru_os`

Комбо на базовом слое:

- `C + V` — переключение на английский;
- `, + M` — переключение на русский.

Макросы отправляют в ОС следующие сочетания:

- английский: `Left Alt + Left Shift`;
- русский: `Left Alt + Right Shift`.

Это означает, что сами русские буквы не печатаются отдельным прошивочным слоем, а выбор языка выполняется на уровне операционной системы.

## Как формируются рисунки раскладки

Рисунок раскладки генерируется автоматически через `keymap-drawer`.

Используются следующие файлы:

- `config/totem.keymap` — описание раскладки;
- `config/totem.json` либо `config/*.json` — описание физической геометрии;
- `keymap-drawer/config.yaml` — настройки отрисовки;
- `.github/workflows/draw_keymaps.yaml` — workflow генерации.

На выходе формируются:

- `keymap-drawer/totem.yaml` — промежуточное представление раскладки;
- `keymap-drawer/totem.svg` — итоговый рисунок раскладки.

---

## Как запускается генерация рисунков

После успешной сборки workflow `build.yml` вызывает reusable workflow:

- `.github/workflows/draw_keymaps.yaml`

Внутри него выполняются действия:

1. checkout репозитория;
2. установка `keymap-drawer`;
3. установка `west`, если требуется загрузка модулей;
4. разбор `config/*.keymap`;
5. генерация `keymap-drawer/*.yaml`;
6. генерация `keymap-drawer/*.svg`;
7. публикация рисунков как артефактов.

---

## Порядок получения прошивки

### Через GitHub Actions

1. Открыть вкладку **Actions**.
2. Запустить workflow **ZMK Firmware**.
3. Дождаться завершения сборки.
4. Скачать артефакты.
5. Выбрать нужный `.uf2` файл:
   - для левой половины;
   - для правой половины;
   - для донгла;
   - для сброса настроек.

---

## Назначение файлов рисунка

### `keymap-drawer/totem.yaml`

Файл содержит машинно-подготовленное описание всех слоёв, подписей клавиш, удержаний, переходов между слоями и других элементов раскладки.

Используется как промежуточный источник для построения SVG.

### `keymap-drawer/totem.svg`

Файл содержит готовое графическое представление раскладки.

Используется для:

- визуальной проверки раскладки;
- публикации схемы клавиш;
- документирования прошивки;
- контроля изменений после редактирования `totem.keymap`.

## Локальная сборка

Пример общей последовательности локальной сборки:

```bash
west init -l config
west update
west zephyr-export
west build -s zmk/app -b xiao_ble//zmk -- -DZMK_CONFIG="$PWD/config" -DSHIELD=totem_left
```

Для других вариантов следует заменить shield:

- `totem_left`
- `totem_right`
- `totem_dongle`
- `totem_dongle dongle_screen`
- `totem_dongle_prospector prospector_adapter`
- `settings_reset`

---

## Примечание по Node.js 20 warnings

Предупреждения GitHub Actions о Node.js 20 относятся к используемым actions и не означают ошибку прошивки.

Для принудительного выполнения actions в Node 24 в workflow уже задано:

```yaml
env:
  FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true
```
